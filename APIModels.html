Multiverse

And API and web framework packed in one thing!

API Models

Only the API deliver you the environment to build a model. The 'application' part of Multiverse, does not support this. We made this decision to force you to build the API as a model and every application should talk with the API to get their data.

Api as a Model

The model part inside the API deliver you the direct connection with the database. We use PDO to create this connection with your database in a safe manner. The current Multiverse model is already generated for you and could you find in the documentation of the database class.

Creating your own model

If you want to create your own model, we highly encourage you to extend the standard Multiverse model. The example below gives you a pattern on how to create your own model.

class MY_Model extends MV_Database {

	public function __construct() {
		parent::__construct();
	}
	
	/* YOUR CUSTOM DATABASE METHODS */
}

Load your newly created model

If you preferred to create a custom model on top of the standard Multiverse model, you should load it in your controller classes which want to use this model.

Use this statement inside your controller to load your new model:

class News extends ApiCore {
	public function construct() {
		Note: this time we do not include the constructor method.
		$this->db = new MY_Model();
	}
}

Setup database connection

Inside the API folder you can find a config.php file. This file is used for some configurations concerning the API. As soon as you use a model, you need to change the database settings inside this file to setup correctly a connection.

Change this part of the config file to create a database connection
$config['database_table'] = '';
$config['database_username'] = '';
$config['database_password'] = '';
$config['database_location'] = ''; // Normally localhost
