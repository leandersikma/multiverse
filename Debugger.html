Multiverse comes with a neat debug tool to simply debug your API, created with Multiverse. To reach this debug tool, go to: 

yourdomain/pathToMultiverse/debug

How to debug

Inside the debugger tool you can post the path to your API class, such as: yourdomain/api/new/get/.

After you have defined the path to the class you want to debug, you could setup the header configurations. At this moment, Multiverse only support GET and POST requests.

If you want to test the class by GET request, use the GET request. Otherwise, you need to config the POST header values.

Config POST request

If you want to test your API class by sending a POST request, you have to send POST values with it. If you choose inside the debugger for POST request, you get an extra textarea which you can fill with data.

You need to know what your API class expects concerning the data. So look at all the $request['key'] parts inside you class function. The only thing that you should do inside the debugger is to fill all those request keys with your values. You should format is as:

key=value&key=value&key=value

An example would be:

title=I'm trying Multiverse!&description=An API centric development framework

Which you can read in your API class function by doing the following:

$request['title'];
$request['description'];

Result debugger

Because Multiverse is using CURL to perform GET and POST requests to and from the API, it is sometimes difficult to simply debug your API class. After you configured the debugger and send the request, you get a result. This result could be everything. Such common results are: JSON response, PHP error, HTML output.

We hope that your experience with building an API is going to be much better by using this debugger.
