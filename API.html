Class: MV_API Extends: AbstractCore

The API class is the class which will be used to route API specific request. The MV_API class extends the AbstractCore class.

MV_API Constructor

MV_API overrides the parent constructor in the following way

public function MV_API() {
	parent::__construct();
	$this->Route();
	$this->sendResponse();
	
}

As you can see we call the Route function first and after all we send the response to the client.

API Configurations Route()

We override the Route() function inside MV_API class, because we have to change the segments. Look at the following example:

In case of a normal request:
example.com/controller/action/parameter/

We get the following segmentation:
example.com/segment:1/segment:2/segment:3/

If we want to get the controller name, we use segment 1.

If we have an API request:
example.com/api/controller/action/paramter/

We have the following segmentation:
example.com/segment:1/segment:2/segment:3/segment:4/

If we want to get the controller name of the API, we have to use segment 2.

After we configured the segments in the way we want, we call the parent routing function so it does the normal logic with the API configurations.

Send response back to the client

In the end the response from the Route() function needs to be send back to the client. Multiverse does this by generating a sendResponse() function to send the data back in a way that is defined for that class.

The sendResponse() function inside MV_API class:

public function sendResponse() {
	header('Content-type: text/json');
	echo $this->output->getOutput();
}
In the near future we are going to implement a builder to form the data as JSON/XML or something.

We output the getOutput() function from the Singleton class Output. This function holds a string with all kind of data that is added trough the URL request.
