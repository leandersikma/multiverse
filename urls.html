Multiverse URLs

Multiverse URLs are designed to be user friendly and optimized for search engines. All the URLs are build from the controller perspective, so here is an example of the URL structure.

example.com/news/item/my_article/

The API part of Multiverse works also with this kind of URL structure. To give an example of an API url:

example.com/api/news/get/

Segments explained

Multiverse URLs are build from the controller perspective. A Multiverse URL is a combination of different segments, as followed:

example.com/controller/action/parameter/

Controller
This is the controller that should handle this request.
Action
The method to use inside the requested controller.
Parameters
Some parameters you could add.

Removing index.php

Every request goes trough index.php. To delete this part of the URI, you need to add the following code to the .htaccess file in the root of the Multiverse folder. 

          	RewriteEngine on
          	RewriteCond %{REQUEST_FILENAME} !-f
          	RewriteCond %{REQUEST_FILENAME} !-d
          	RewriteRule ^(.*)$ index.php?$1 [L]
