Application Views

Views are used to generate the markup that should be shown to the user. Views can be used to print a whole HTML page or a fragment of a HTML page, like header/content and footer views. Views does not have logic inside itself, it contains only printable markup and a litte bit PHP but only for printing data.

Example of a View

A view can be as simple as a standard HTML page. 
<html>
	<head>
		<title>View</title>
	</head>
	<body>
		This is a view.
	</body>
</html>
Let's call this file: view.html

Place this file inside application/views/

Loading view inside controller

To load a view inside a controller, you have to use the following function:

$this->load->view('view');

Here is an example of a controller of the aforementioned view file:

class MY_Controller extends MV_Controller {
	
	public function __construct() {
		parent::__construct();
	}
	
	public function index() {
		$this->load->view('view');
	}
}

Loading multiple views

If you want to load multiple views in a controller, you could use the load->view function multiple times: 
$this->load->view('header');
$this->load->view('content');
$this->load->view('footer');

Adding data to the view

If you have dynamic content inside your view, you want to send data from the controller to the view. The view() function has two parameters, described as follow: 

$this->load->view($view, $args = null);

If you want to send data from a controller:

$data['list'] = array('List 1', 'List 2', 'List 3');
$this->load->view("listView", $data);

Inside the list view you can access the list by using the variable: $list

		
<html>
	<head>
		<title>View</title>
	</head>
	<body>
		<?php 
			foreach($list as $list) {
				echo $list;
			}
		?>
	</body>
</html>
